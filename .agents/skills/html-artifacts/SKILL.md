# HTML Artifacts

Use this skill for standalone visual artifacts, prototypes, or documentation previews that should not become app features.

## Rules

- Do not place experimental artifacts inside production routes.
- Keep artifacts clearly labeled as non-product.
- Use semantic HTML, responsive CSS, and accessible contrast.
- Prefer actual app implementation when the user asks for a working feature.

## Good Uses

- UX comparison sketches.
- Architecture diagrams exported as HTML.
- Visual review aids for docs.

## Avoid

- Treating HTML artifacts as production UI.
- Adding app dependencies for one-off artifacts.
