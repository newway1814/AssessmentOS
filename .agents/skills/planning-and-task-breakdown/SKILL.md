# Planning And Task Breakdown

Use this skill when the work is too broad for a single obvious edit.

## Plan Format

- Goal.
- Constraints and non-goals.
- Files or modules likely affected.
- Ordered slices.
- Verification commands.
- Risks or open questions.

## Good Slices

- Specs/docs update.
- Domain schema and helper tests.
- Mock adapter.
- Single route or workflow.
- UI polish and browser verification.

## Rules

- Keep PR-sized units small even when no PR is created.
- Do not plan hidden implementation of future scope.
- Prefer implementation steps that can each be verified.
