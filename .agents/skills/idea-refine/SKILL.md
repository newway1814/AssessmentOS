# Idea Refine

Use this skill when the user has a product idea that is not ready for implementation.

## Refine

- Primary user: teacher, academic coordinator, reviewer, or school admin.
- Job to be done.
- Current workflow pain.
- MVP behavior.
- Non-goals.
- Data and rights implications.
- Acceptance criteria.

## Output

Keep the result practical:

- One-paragraph problem statement.
- Core workflow.
- In-scope / out-of-scope bullets.
- Open questions only where unavoidable.

## Rules

- Do not overbuild.
- Prefer the next smallest usable workflow.
- Update `/specs` when refined behavior becomes product direction.
